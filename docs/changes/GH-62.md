# GH-62: Reuse the CLI session across same-stage retries

## What shipped

`invoke_stage` captured the CLI's `session_id` but discarded it, so a retry of
the same stage always started a cold process and re-read the codebase at full
price. This change threads `session_id` back into the next same-stage
invocation as `--resume <id>`, so a retry's prior reads become cheap
prompt-cache hits instead of a cold re-read.

## Surface changes

### CLI / workflows

- `adw/invoke.py`: `build_command(stage, model, claude_bin=None,
  resume_session_id=None)` appends `["--resume", resume_session_id]` to the
  argv only when an id is given.
- `adw/invoke.py`: `invoke_stage(..., resume_session_id=None)` forwards the id
  into `build_command` only on the non-isolation (host) path — an isolated run
  gets a fresh container per stage, so there is no session store to resume.
- `adw/workflow_runner.py`: `_make_invoke`'s closure now holds a
  `sessions: dict[str, str]` keyed by stage name. Each `invoke(stage, ...)`
  call passes `sessions.get(stage)` as `resume_session_id` and stores
  `result.session_id` back into the map only when it is truthy (so a crashed
  attempt with no session id never clobbers the last good session).

## Behavior & breaking changes

None observable externally. A same-stage retry now resumes the prior CLI
session instead of starting cold; a first attempt, an unseen stage, a
different model, or an isolated run is unaffected and still starts cold.

## How it was verified

  model[+tools] match" → `tests/test_invoke.py::test_build_command_resume_appends_flag_only_when_given`
  (asserts `--resume s1` present when given, absent when not) and

- "A fresh stage, a different model, or a first attempt still starts cold." →

- "Breaker / iteration accounting is unchanged." → diff is scoped to argv

  `adw/safety.py`, `adw/state.py`, and `run_ticket` are untouched in this

- "Full suite stays green (`uv run pytest -q`)." → test stage ran the full

## Review notes

- Reuse is keyed by stage name, which by construction is the same model and
  same tool scope, so a resumed session is always interchangeable — no
  separate model/tools check was needed.
- The isolation path is deliberately excluded (documented in `invoke_stage`'s
  docstring): each isolated stage runs in a fresh container, so there's
  nothing to resume there.
- This run found the implementation already complete and committed from a
  prior iteration that was interrupted mid-test-stage by a session-limit
  message (not a code failure); this iteration only re-verified the existing
  tree and re-ran the suite the limit had interrupted.
- The branch is stacked on unmerged prior work (`poll_all.py`, `github.py`,
  `docs/changes/GH-56.md`, etc. all show up in the diff to `main`); merge
  against this ticket's actual parent commit, not `main` directly.
