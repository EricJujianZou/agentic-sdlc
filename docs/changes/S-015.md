# S-015 — Quota-limit interruptions become resumable (`quotad`), not blocked

## What shipped

A usage-limit halt is now a distinct, auto-resumable `quotad` state instead
of being lumped into `blocked` (human-gated). The breaker classifies a
genuine quota cut-off (not a stage that merely talks about usage limits),
sets a cooldown from the reset time embedded in the halt text when present
(falling back to a configurable default otherwise), the orchestrator
surfaces `quotad` as its own outcome, `_finalize_story` skips the PR and the
observer for it, and the picker treats `quotad` stories as eligible to
auto-resume.

## Behavior & breaking changes

- `STORY_STATUSES` gains `quotad` (`adw/tickets.py:14`); existing statuses
  are unchanged.
- `CircuitBreaker._evaluate` only matches `detect_usage_limit` when the
  stage result did **not** succeed (`adw/safety.py`). Previously it scanned
  every result's raw output, so a stage whose subject is usage limits
  (e.g. this very ticket) could trip its own circuit on a successful run —
  this is the dogfood false-positive from 2026-06-19 and is now fixed.
- `CircuitBreaker.record` no longer applies the generic
  `circuit_cooldown_minutes` to a usage-limit halt. It instead uses
  `_parse_usage_reset` (epoch-suffix or ISO-8601 timestamp in the halt
  text) when a future timestamp is present, else
  `SafetyConfig.usage_limit_cooldown_minutes` (default 300, configurable
  via `configs/budgets.json`).
- `orchestrator.run_ticket` returns outcome `"quotad"` instead of
  `"halted"` when the breaker's halt reason is exactly
  `USAGE_LIMIT_HALT_REASON`; every other halt still returns `"halted"`.
- `workflow_runner._finalize_story` early-returns for a `quotad` outcome:
  marks the story `status="quotad"`, swaps the `in-progress` label for a
  new `quotad` label, and does **not** open a PR or invoke the observer.
  The done/blocked/halted branches are otherwise unchanged.
- `tickets.pick_next_story` / `pick_next_stories` now select
  `status in ("open", "quotad")` instead of just `"open"`, so a
  quota-interrupted ticket resumes automatically once `check_cooldown`
  clears it. `"blocked"` stays excluded (human-gated).
