# GH-63: Right-size and gate the expensive opus stages

## What shipped

`decompose` and `observe` both ran on `claude-opus-4-8`, but neither needed
opus on every call: decompose only expands a terse ticket into acceptance
criteria, and the observer ran a whole-repo opus diagnosis on *every*
non-`done` outcome, including failures that are clearly ticket-level on their
face. This change moves `decompose` to sonnet and adds a two-tier gate in
front of the observer's opus pass.

## Surface changes

### Config

- `configs/models.json`: `decompose` is now `"sonnet"`; added
  `"observe_triage": "sonnet"`. `observe` stays pinned to
  `claude-opus-4-8` — it is now the *escalation* pass, not the default one.

### Observer triage/escalate

- `adw/invoke.py`: added `"observe_triage"` to `STAGE_TOOLS` with the same
  read-only scope as `"observe"`.
- `adw/workflow_runner.py`: `compose_stage_prompt` now resolves a
  `base = _PROMPT_BASE_STAGE.get(stage, stage)` stage name and uses `base`
  for the command file, the stage spec, and the `state.stage` written into
  the ticket context. `_PROMPT_BASE_STAGE = {"observe_triage": "observe"}`,
  so the triage prompt is byte-identical to the observe prompt — one
  classification bar, no duplicated prompt asset. Model lookup and the saved
  prompt/output filenames still key on the real stage name (`observe_triage`),
  so the two passes never collide on disk.
- `adw/orchestrator.py`: added `OBSERVE_TRIAGE_STAGE = "observe_triage"` and
  factored the per-pass invoke+log+parse logic into `_run_observer_pass`.
  `run_observer` now runs the sonnet triage first; it only invokes the opus
  `OBSERVE_STAGE` pass — and treats that result as authoritative — when
  triage classifies the failure as `"harness"`. A ticket-classified or
  unparseable triage returns unchanged with no second invoke.
- `adw/workflow_runner.py`: `_observe_and_report` now short-circuits before
  calling the observer at all when `failure_reason` matches
  `_is_ticket_level_failure` — a conservative, named substring list
  (`permission denial`, `permission denied`, `circuit open`, `test failed`,
  `tests failed`, `pytest`, `assertionerror`). On a match it posts the
  existing ticket-level clarify comment (`CLARIFY_LABEL`) directly, with no
  agent invocation at all.

## Behavior & breaking changes

- Decompose runs cheaper; its output contract (`criteria` list) is unchanged.
- A ticket-level failure that doesn't match `_is_ticket_level_failure` still
  gets a sonnet triage pass; only a triage that calls it `"harness"`
  escalates to opus. A pre-skipped failure gets zero agent invocations.
- `status.py`'s stage-name check only requires the emitted `"stage"` value to
  be in `STAGES`; since the triage prompt is the unmodified `OBSERVE.md`, it
  still emits `"stage":"observe"`, which validates without any contract
  change to the agent or to `STAGES`.

## How it was verified

- "Decompose uses a non-opus model." → `configs/models.json` `decompose ==
  "sonnet"`.
- "The observer does not spend opus on a clearly ticket-level failure; it
  escalates [only when harness-level]" →
  `tests/test_observer.py::test_run_observer_success` (harness triage: two
  invokes, `["observe_triage", "observe"]`),
  `test_run_observer_ticket_triage_does_not_escalate` (ticket triage: one
  invoke, no escalation), `test_run_observer_reports_problem_on_non_success`
  (non-success triage short-circuits, no escalation), and
  `test_observe_and_report_skips_observer_for_ticket_level_reason` /
  `test_is_ticket_level_failure` (pre-skip never calls `run_observer`).
- "Self-host behavior is otherwise unchanged; full suite stays green." →
  `uv run pytest -q`: 369 passed.

## Review notes

- The pre-skip heuristic in `_is_ticket_level_failure` is deliberately narrow
  and kept in one named tuple so it's trivially tunable. It includes
  `"permission denial"`/`"circuit open"`, even though guard/permission
  denials have surfaced real harness bugs before (see
  `dogfood-usage-limit-false-positive`); the tradeoff is intentional per the
  ticket's own framing (a guard denial with a stated `failure_reason` is
  treated as ticket-level), but if a future run misses a real harness bug
  hiding behind a permission denial, narrowing this list is the fix, not
  removing the gate.
- Rejected: threading a `model=` override through the `invoke(stage, state,
  story)` closure to avoid a pseudo-stage. It would force a signature change
  across every injected test invoke and leak the `models` dict out of
  `_make_invoke` to `run_observer`. The pseudo-stage keeps `models[stage]`
  resolution local and changes no signatures.
- Rejected: a dedicated lighter triage prompt instead of reusing `OBSERVE.md`.
  Genuinely cheaper per call, but duplicates the classification bar and risks
  drift between the two prompts; reuse via `_PROMPT_BASE_STAGE` is simpler.
