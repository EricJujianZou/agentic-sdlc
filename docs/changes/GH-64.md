# GH-64: Budget-aware blocker handling

## What shipped

Two budget-saving guardrails for the stage pipeline. Read-only stages
(plan/decompose/review) now stop and report `blocked` immediately on a
hard structural blocker instead of producing the full deliverable first.
Separately, the implement prompt now warns non-`system-repair` tickets up
front that the PreToolUse guard denies writes under harness dirs, so
implement reports `blocked` instead of attempting a known-denied `Write`.

## Behavior & breaking changes

- `commands/PLAN.md`, `commands/DECOMPOSE.md`, `commands/REVIEW.md` each
  gained one sentence appended to the existing headless-rule paragraph:
  "On a hard structural blocker, report `blocked` immediately — do not
  spend tokens producing the full plan/criteria/review first."
- `adw/workflow_runner.py` `compose_stage_prompt`: for `stage == "implement"`
  (resolved via `base`) and `story.type != "system-repair"`, the composed
  prompt now gets an appended "## Harness-edit policy" section naming the
  guarded dirs (`adw/`, `hooks/`, `workflows/`, `stage_specs/`, `skills/`,
  `commands/`, `configs/`, `plans/`, `.claude/`) and instructing the agent
  to report `outcome: "blocked"` rather than attempt the write.
  `system-repair` tickets receive no such section, since they're allowed
  to edit those dirs.
- New module constant `_HARNESS_DIRS` in `adw/workflow_runner.py:543`
  duplicates `hooks/pretooluse_guard.py`'s `HARNESS_DIRS` list — `hooks/`
  has no `__init__.py`, so it isn't importable from the engine. A comment
  on the constant names the guard as source of truth.

## How it was verified

- AC1 (fast-stop on hard blocker) → `test_fast_stop_rule_present_in_readonly_stage_commands`, parametrized over PLAN/DECOMPOSE/REVIEW, asserting the new sentence is present in each file.
- AC2 (type-conditional implement policy) → `test_compose_stage_prompt_implement_harness_policy_by_type`, asserting the "Harness-edit policy" section (and a dir name like `workflows/`) is present for a `feat` story's implement prompt and absent for a `system-repair` story's.
- AC3 (suite green) → test stage ran `uv run pytest -q`: "373 passed in 15.86s".

## Review notes

- `_HARNESS_DIRS` is a byte-identical duplicate of the guard's list; if the
  guard's `HARNESS_DIRS` changes, this copy must be updated too. Flagged in
  code as known debt rather than solved via cross-import (would require
  adding `hooks/__init__.py`, which is out of scope for this ticket).
- The fast-stop sentence was added only to the three read-only stages
  (PLAN/DECOMPOSE/REVIEW); IMPLEMENT/TEST were intentionally excluded since
  they aren't read-only deliverable-producing stages in the same sense.
