# GH-83 — Blocked outcomes no longer open a PR

## Problem

A run that ended `blocked` could still open a PR. `_notify_github` opened one
when `outcome == "done" OR _branch_has_reviewable_diff(branch)` — the GH-59/#69
rule that surfaces partial work on a blocked ticket. GH-61's blocked re-run hit
this: squash-merged work registered as a fresh "reviewable diff", so it opened
spurious PR #77. The owner prefers that a blocked outcome never surface a PR.

## Change

- `adw/workflow_runner.py`
  - `_notify_github` opens/updates a PR only when `outcome == "done"`. Any
    non-done outcome (blocked/halted/quotad) opens no PR; the branch is still
    pushed and the outcome comment still fires, so nothing is lost and the
    phone is still notified.
  - Removed the now-orphaned `_branch_has_reviewable_diff` helper and the
    `_BOOKKEEPING_PATHS` constant it used.
- Tests updated to the new contract (blocked → no PR + comment; done → PR);
  removed the two direct `_branch_has_reviewable_diff` unit tests.

## Reversal note

This **reverses GH-59/#69**, which deliberately opened a PR for a blocked
ticket that produced partial work so a human could review it. Under GH-83 that
partial work still lives on the pushed `adw/<id>` branch — it just no longer
auto-opens a PR. Flagged here and in the PR for review at the merge gate.
