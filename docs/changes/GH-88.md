# GH-88: poll_once log hides which issue sync skipped and why

## What shipped

`sync_fn` in `poll_once.main()` previously discarded the per-issue `(sid, reason)`
pairs from `pull_and_sync` and logged only a bare count (`1 skipped`). This change
introduces `format_sync_message()` in `poll_once.py` that surfaces non-routine skips
(anything other than `"already synced"`) by name and reason in the poll-log summary
line, while keeping routine already-synced counts noise-free. A new constant
`ROUTINE_SKIP_REASON` in `sync_issues.py` is the single source of truth for that
sentinel.

## Surface changes

### CLI / workflows

**`format_sync_message(added, skipped)`** — new public function in `workflows/poll_once.py`.

- Builds the sync summary string for one poll pass.
- `added`: list of newly synced stories.
- `skipped`: list of `(sid, reason)` tuples from `pull_and_sync`.
- Non-routine skips (reason ≠ `ROUTINE_SKIP_REASON`) are appended as
  `(GH-87: expected exactly one type label ...)` detail.
- Routine "already synced" skips are counted but not named.
- The returned string passes through the existing `_clip` / `_LOG_FIELD_CAP`
  path in `format_summary_line`, so the one-line invariant is preserved.

Example output (before vs after):

```
# Before
sync: synced: +0 new story(ies), 1 skipped

# After
sync: synced: +0 new story(ies), 1 skipped (GH-87: expected exactly one type label ... got ['bug', 'system-repair'])
```

**`ROUTINE_SKIP_REASON`** — new module-level constant in `workflows/sync_issues.py`.

- Value: `"already synced"`.
- Used internally by `sync_issues()` and imported by `poll_once.py` and the test
  suite to avoid hardcoding the sentinel string in multiple places.

## How it was verified

| Acceptance criterion | Test evidence |
|---|---|
| Non-routine skip IDs and reasons appear in `format_summary_line` output | `test_format_sync_message_non_routine_skip_named_in_log` — asserts `"GH-87"` and `"expected exactly one type label"` in the formatted line |
| Routine `"already synced"` skips counted but not named | `test_format_sync_message_routine_skip_counted_but_not_named` — asserts `"1 skipped"` present, `"GH-5"` absent |

| `uv run pytest -q` stays green | 382 passed in 31.21s |
