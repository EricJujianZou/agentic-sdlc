# GH-87: skip in-flight adw issues during the multi-repo sweep

## What shipped

The `poll_all` sweep now skips any issue whose work is already in flight — an
open PR with head `adw/<id>` or an existing `adw/<id>` branch on origin — rather
than re-working it from scratch in a fresh clone. A shared helper `in_flight_ref`
in `adw/github.py` performs the check; `pick_next_story` gains an `exclude`
parameter so already-in-flight ids are filtered without touching the picker's
pure, network-free contract. If a repo's top story is in flight, its next-priority
story is still considered (no starvation).

## Surface changes

### CLI / workflows

**`in_flight_ref(owner, repo, ticket_id, token, repo_root=None) -> str | None`**
(`adw/github.py`, after `open_or_update_pr`)
- Returns `"PR #N"` if an open PR with `head=adw/<ticket_id>` is found (reuses the
  `pulls?state=open&head=` query already used by `open_or_update_pr`).
- Returns `"branch adw/<ticket_id>"` if `git ls-remote --heads origin adw/<ticket_id>`
  finds the ref.
- Returns `None` if neither is true.
- Fails open on `GitHubError` or subprocess error — a failed check never blocks
  an otherwise-valid pick.

**`pick_next_story(prd, *, types=..., exclude: set[str] | None = None)`**
(`adw/tickets.py`)
- New keyword-only `exclude` param drops story ids present in the set.
- Defaults to `None`, preserving all existing call sites unmodified.

**`sweep()` changes** (`workflows/poll_all.py`)
- Builds `repo_slugs: dict[Path, tuple[str, str]]` during the clone/sync phase.
- Adds `in_flight_skips: dict[Path, set[str]]` before the work loop.
- After picking a story, calls `in_flight_ref`; if non-None:
  - Prints `skipped: <id> already in flight (<ref>)`.
  - Adds `<id>` to that repo's `in_flight_skips` set.
  - Re-appends the repo to `active_repos` so lower-priority stories still run.

## Behavior & breaking changes

`pick_next_story` now accepts an `exclude` keyword argument. Existing call sites
that pass no keyword arguments are unaffected. Any caller using
`pick_next_story(prd)` or `pick_next_story(prd, types=...)` compiles and runs
identically. Call sites that monkeypatched `pick_next_story` as a positional
lambda (e.g. `lambda prd: ...`) were updated in the test suite to
`lambda prd, **kwargs: ...` to absorb the new keyword argument.

## Review notes

- The branch is stacked on unmerged GH-56, so `git diff main...HEAD` shows ~2 000
  lines including all of `workflows/poll_all.py` (new in GH-56). GH-87's own
  contribution is ~90 lines across the three source files plus tests.
- `poll_once`'s single-repo path is intentionally left unwired. On a persistent
  checkout, a completed story is already marked `done` and never re-picked, so
  the bug documented in this ticket does not exercise that path. `in_flight_ref`
  is importable there if needed later.
- Each sweep pick now makes one extra `GET /repos/.../pulls` call plus one
  `git ls-remote` per picked story. Both are bounded by the iteration cap and
  are the intended cost of the guard.
