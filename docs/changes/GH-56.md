# GH-56: Multi-repo driver — auto-discover, auto-clone, sweep all adw backlogs until quota

## What shipped

An unattended multi-repo sweep driver, `workflows/poll_all.py`, built on top of
the repo-agnostic engine (S-011). One invocation discovers every account repo
with at least one open `adw`-labeled issue, auto-clones any that aren't on
disk yet, syncs each into its own `prd.json`, then works tickets round-robin
one-per-repo until every backlog is empty or the account-wide quota trips. The
existing circuit breaker (`adw/safety.py`) is untouched — the driver only
reads the per-repo cooldown it already persists and copies it into a new,
out-of-repo global store.

> Note: this branch is stacked on the unmerged GH-60 commits (`adw/safety.py`
> sign/clamp fix, `docs/changes/GH-60.md`), which show up in `git diff
> main...HEAD` but are not GH-56's work — see File map.

## CLI / workflows

`workflows/poll_all.py` — `uv run python workflows/poll_all.py [--max-iterations N]`.
Single-flight guarded (`sweep_lock_path`, account-wide, unlike `poll_once`'s
per-repo lock). On each run:

1. Checks the global cooldown store; if active, exits immediately with that reason.
2. `discover_targets` — lists the account's own repos (`adw/github.py:list_account_repos`,
   new) and keeps those with an open `adw` issue (`list_adw_issues`, reused).
3. `ensure_clone` per target — engine's own repo short-circuits to
   `paths.engine_root()` (no clone); others clone if missing, else
   fetch+checkout+ff-only-merge `main` in a managed dir.
4. `pull_and_sync` (reused) per repo to refresh its `prd.json`.
5. Round-robins `run_one_story`/`reap_stale_in_progress`/`pick_next_story`
   (all reused) across active repos via a per-repo `_target` context manager
   that flips `ADW_REPO` and clears `paths.target_root`'s `lru_cache`,
   restoring both on exit (including on exception).
6. On a `quotad` outcome: copies that repo's already-parsed
   `state.cooldown_until` into `write_global_cooldown` and halts the whole
   sweep. Otherwise continues until every repo's backlog is empty.

## Schemas & data

- `adw/github.py: list_account_repos(token, owner) -> list[(owner, name, clone_url)]`
  — pages `/user/repos` (`affiliation=owner`, `per_page=100`), filters to repos
  whose `owner.login` matches, excluding `archived`/`disabled`.
- Global breaker file (new, out-of-repo): `%LOCALAPPDATA%/adw/breaker.json`
  (override `ADW_GLOBAL_BREAKER`) — `{cooldown_until, reason, written_at}`,
  written atomically (temp file + `fsync` + `os.replace`), read fail-open (any
  parse error means "no cooldown").
- Managed clone dir (new): `%LOCALAPPDATA%/adw/repos/<owner>/<name>`
  (override `ADW_REPOS_DIR`).
- Sweep lock (new): `%LOCALAPPDATA%/adw/locks/poll-all.lock` (override
  `ADW_SWEEP_LOCK`), reusing `adw/locks.single_flight`.

## Review notes

- Branch is stacked on unmerged GH-60 — merge GH-60 first, or land both
  together; `adw/safety.py` and `docs/changes/GH-60.md` are not part of this
  ticket's diff despite appearing in `git diff main...HEAD`.
- `_ENGINE_SLUG` is resolved once at import time against the engine's own git
  remote, not lazily inside `ensure_clone` — a test that monkeypatches
  `subprocess.run` to fake a different repo's git calls would otherwise also
  stomp the real self-host check (see `progress.txt`, GH-56 entry).
- Sequential v1 only — parallel cross-repo execution is deferred until the
  unmerged #4 worktree work lands.
- Auto-discover is intentionally allowlist-free: any repo ever labeled `adw`
  joins the sweep, including the engine's own.
