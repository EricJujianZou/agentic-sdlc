# GH-16 — Dashboard: ticket count badge on each status column

## What shipped

Each dashboard column header (open / in_progress / blocked / done) now shows
a styled numeric badge with the count of cards in that column, replacing the
previous inline `(N)` text appended to the label.

## Surface changes

### Components / pages

- `.column-header` in `dashboard/index.html` — now a flex container
  (`display: flex; align-items: center; justify-content: space-between;`)
  holding two children: a label `<span>` (status name) and a
  `<span class="column-count">` (the count), instead of one `textContent`
  string.
- `.column-count` (new CSS rule, `dashboard/index.html`) — small pill badge:
  `#334155` background, `#f1f5f9` text, `0.7rem` font, rounded corners,
  `margin-left: 0.4rem`, `text-transform: none` (opts out of the header's
  uppercase transform).

## Behavior & breaking changes

The header no longer renders `LABEL (N)` as a single text string; the count
is now a separate DOM element. Any external code or test relying on the
header's `textContent` matching that exact string would break — no such
dependency exists in this repo (`tests/test_dashboard.py` only checks file
existence and a `prd.json` substring).

## How it was verified

- "Each column header shows a numeric badge with the count of cards in it"
  → `render()` builds a `span.column-count` per column from
  `buckets[status].length` for every entry in `STATUSES`
  (`dashboard/index.html`).
- "The badge updates correctly when prd.json has zero cards in a column
  (shows 0)" → `headerCount.textContent = String(buckets[status].length)` is
  set unconditionally (no zero-guard), so an empty bucket renders `0`.
- "tests/test_dashboard.py still passes and the full suite stays green" →
  test stage ran the full suite, 209 passed; `test_dashboard.py` asserts
  only file existence and a `prd.json` substring, neither touched by this
  diff.
